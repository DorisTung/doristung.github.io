Custom Dashboard

Custom dashboard provides a summary view which allows security operation team to select and arrange the widgets based on their needs and roles.

Role & Team
My Role
 UX designer
Key Skills
 Use cases study, Concept ideation, Wireframing
Design team
 Visual Designer, Technical writer
Development team
 Product manager, Project manager, Back-end engineer, Front-end developer, QA engineer
Design goal

Clarify the use cases for the custom dashboard and design a user flow that could help Security Operation Center (SOC) members to easily arrange the content on the dashboard.

Design process
Define use case

The use cases for custom dashboard are really widely, from the dashboard structure down to the widget contents all could be customized. In the first stage, we focus on explore and collect the possible use cases to help the team build the consensus of the long-term goal while also discuss the priority and define the MVP scope

Highlight:

  • Each role in the SOC team will select different widgets based on the responsibility.
  • Users need to add, remove, arrange the widgets and even edit the data.
  • Users need to drill down from the dashboard widget to understand the detail of the data.
Concept ideation and Wireframe

Create the end to end user flow and wireframe based on the project plan. Besides, since there are more new widgets being updated, I also create the UX guideline of the widget framework to ensure UX designers could follow it and make the design consistency.

Highlight:

  • Each user account can create its own dashboard view.
  • Users can add the widgets from the widget category and move or remove them from the dashboard. Also able to edit the data range on each widget.
  • Users can click the “Go to app” button on the widget to drill down to the detail information.
Design iteration

After feature released, collaborate with PM and data analyst to keep tracking the user behavior and collect user feedbacks. For example, we found that the usage of new released widgets is much lower than others and assume that it’s hard for users to notice those new widgets. Then we refine the design and user flow based on the feedbacks.

Highlight:

  • Highlight the “Widget catalog” button while there is a new widget released.
  • Set Unused widgets on the top of the widget catalog to encourage users to use them.
Design outcome
  • Users are able to select widgets from the widget catlog on their own with different criteria and combination.
  • Users can freely order the widget while also create multiple pages for different view.

As a SOC manager, he/she could create a dashboard that shows the overall status of alert handling, so that he/she can know whether the team has enough resource.

As a Security Analyst, he/she can have a summry view to understand how many alerts he / she need to handle and can quickly drill down to the detail.
