Network Analytics Report

A network analytics report provides an interactive Sankey digram to help users understand every step of the attack. It provides the network traffic and correlate events in a graph for real-time visibility and let user have a bigger picture of the full attack.

My Role
 UX designer
Key Skills
 User research, Concept ideation, Wireframing
Team
 UX researcher, Product manager
Design goal

Enhance the usability of the network analytics report based on the user interview findings. Aim to provide a an interactive report which is more clear and easier to use.

Design process
  • Co-work with the user researcher to conduct user interviews, including define the research goal, research questions and create the materials for testing.
  • Prioritize the user needs that we collected from the user research and came up with the UI design accordingly.
  • Conduct a quick validation with the previous interviewees to make sure the design concept indeed solve their problem.
  • Enhance the usability of the network analytics report based on the user interview findings. Aim to provide a an interactive report which is more clear and easier to use.
Design outcome
1.Enhance the summary view
#Pain points
  • Not be able to see the overall attack pattern and impact scope at a glance
  • Lack of connection between the summary content and the graph which make the report hard to understand.
  • The information is flat without any prioritization and hence make it hard for users to get the point.
#New design
  • Provide the report title and impact scope on the top that user can catch it once they view the report.
  • Provide “focus” function on the object in summary which helps user to indicate to the related node on the sankey chart.
  • Highlight the trigger event and differentiate the time step with other information to make it easier to understand.
2. Adjust the report layout
#Pain points
  • The network analytics report is not easy to read because the content is too long and it is split into three parts without connection.
#New design
  • Fix the report title on the top and make the summary expandable so that users can read the summary as a guidance at first and hide it when they need to dig into the graph.
  • Show the detail information event right beside the graph that users could compare it with the graph.
3. Make the report interactive
#Pain points
  • The report content might get too huge and unable to view in one full screen.
  • Some information are concluded as not important but still take up the space.
  • The sankey chart get hard to read when there are too many lines on it.
#New design
  • Provide a filter function that allows users to narrow down the scope of the report.
  • The filter not only reduce the sankey chart but also narrow the detail information so that users could focus on the part they are interested in or needed to take care.
